Adding Hermes alongside Claude

Marginalia's AI (chat, summarize, auto-tag, claim-verify) runs through the Claude Agent SDK today. Nous Research's Hermes is an open-weight model you can run locally — fully private and offline. Here's how they trade off, and where each fits.

At a glance

C

Claude (current)

Opus 4.8 · Sonnet 4.6 · Haiku 4.5 — cloud API via Agent SDK
  • Frontier reasoning & long-context synthesis
  • Robust agentic tool-use loop (reads PDFs itself)
  • Up to ~200K–1M token context
  • Zero setup beyond sign-in / API key
  • Per-token cost · needs network · data leaves the machine
H

Hermes (proposed)

Nous Hermes 3/4 (Llama-based) — local via Ollama / vLLM, OpenAI-compatible
  • Runs 100% locally — private & offline
  • Free after hardware (or cheap via OpenRouter)
  • Open weights — inspectable & fine-tunable
  • Good at summarize / tag / extract / library Q&A
  • Needs a GPU/RAM · weaker frontier reasoning · you own the tool loop

Why add Hermes

Privacy as a real feature

Sensitive / unpublished papers can be summarized and tagged with nothing leaving the laptop — squarely on Marginalia's local-first brand.

Free bulk processing

Auto-tag and embed-summarize an entire library overnight with no API bill — exactly the high-volume, lower-stakes work local models are good at.

Offline & resilient

Works on a plane or behind a strict firewall; no account, no rate limits, no outage.

Choice & trust

Open weights users can inspect; an escape hatch if cloud pricing or policy changes.

The tradeoffs

Hardware burden

A useful Hermes (8B–70B) wants a modern GPU or Apple-silicon with enough RAM; small machines get slow, lower-quality output.

Weaker agentic PDF reading

Claude's Agent SDK drives the Read-tool loop reliably. With Hermes you implement tool-calling + the read→answer loop yourself; expect more misfires.

Two code paths to maintain

The sidecar must branch: Agent SDK for Claude vs an OpenAI-compatible HTTP client for Hermes — more surface area, more testing.

Quality ceiling

For hard cross-library synthesis and claim verification, frontier Claude still leads; Hermes is "good enough" for routine tasks.

Recommendation

Hybrid: Claude by default, Hermes as the private/offline engine

Keep Claude as the default for chat, claim-verification and agentic PDF reading (where frontier reasoning + the tool loop matter). Add Hermes as a selectable provider for the routine, high-volume, privacy-sensitive jobs — summarize, auto-tag, extract, library Q&A — routed through a local OpenAI-compatible endpoint. The user picks per the model dropdown that already exists; their text only leaves the machine when they choose a cloud model.
